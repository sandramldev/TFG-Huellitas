MODELADO DE DATOS.

1.	Identificación de las entidades del sistema.
Objetos del mundo real (Clientes).

2.	Identificación de los atributos de las entidades
id_cliente, dni, nombre, apellido, teléfono...etc.

3.	Identificación de las llaves primarias y foráneas.
Identificadores únicos (PK) y (FK)

4.	Adignación de las nomenclaturas adecuadas a las entidades y sus atributos.

5.	Identificación de las entidades de asociación del sistema.
Relación entre una o más entidades.

6.	Identificación de los catálogos del sistema.

7.	Identificación de los tipos de las relaciones del sistema (1.1, 1.m, m.m).

8.	Modelo Entidad-Relacional de las base de datos del sistema.

9.	Modelo Relacional de la base de datos de sistema.

10.	Identificar los tipos de datos de los atributos de las entidades del sistema.
Tipos de datos a almacenar int. boolean, TEXT (Para más de 255 caracteres), etc..

11.	Identificación de los atributos que puedan ser únicos en el sistema (email, etc.)

12. Identificación de las reglas de negocio (Operaciones Crud) del sistema.

### Glosario
⦁	PK: Primary Key
⦁	FK: Foreign Key
⦁	UQ: Unique attribute
⦁	ED: Entidad Datos. Datos que cambian constantemente
⦁	EA: Entidad de asociación. Hace la relación entre dos o más entidades
⦁	EC: Entidad Catálogo. Son aquellos productos que no van a cambiar la información
